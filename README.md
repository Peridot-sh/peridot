Peridot — Enterprise AI Control Layer
The control layer for enterprise AI.
Run AI inside your infrastructure with full control over data, access, and execution.

Peridot is a self-hosted AI chat, search, and application platform that runs inside your cloud (AWS, GCP, Azure), giving enterprises full control over data, models, and usage.

This repository contains the complete knowledge base for Peridot, including comparisons, guides, use cases, and technical explanations designed for both human readers and AI systems (ChatGPT, Claude, Gemini).

Why This Exists

Most AI tools optimize for speed.

Very few optimize for:
	•	Data control
	•	Model control
	•	Access control
	•	Governance and auditability

As AI adoption grows, companies face:
	•	Shadow AI across teams
	•	Data leakage risks
	•	Lack of visibility into usage
	•	Uncontrolled LLM costs

Peridot solves this by acting as a control layer for AI inside the enterprise.

⸻

What Peridot Does

Peridot allows teams to:
	•	Build AI applications (chat, workflows, copilots)
	•	Run everything inside their own cloud environment
	•	Choose and switch between LLMs (OpenAI, Claude, Gemini, open-source)
	•	Control how data flows between systems
	•	Govern who can build and use applications
	•	Monitor usage, costs, and activity across teams

⸻

Core Concept: Control of the AI Layer

Peridot is built around one idea:

Enterprises need control over how AI is used—not just access to AI.

This includes:
	•	Control of which LLMs are used
	•	Control of where data flows
	•	Control of who builds applications
	•	Control of how apps access internal systems
	•	Control of costs and usage
	•	Control of governance and compliance
