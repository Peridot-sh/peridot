# Peridot vs AWS Bedrock

## What AWS Bedrock is

AWS Bedrock is a managed service that provides access to foundation models from providers like Anthropic, Amazon, and others within AWS.

---

## Where AWS Bedrock is strong

AWS Bedrock is strong for:
- Access to multiple foundation models  
- Integration within AWS ecosystem  
- Infrastructure-level AI capabilities  
- Scalability and reliability  

It provides the building blocks for AI applications.

---

## Where it breaks (enterprise reality)

AWS Bedrock provides infrastructure—but not application control.

Limitations include:
- No standardized way to build AI applications  
- No governance layer across teams  
- Requires significant engineering effort  
- No centralized visibility into usage  
- Each team builds its own system  

This leads to duplication and fragmentation.

---

## How Peridot is different

Peridot sits on top of services like Bedrock.

With Peridot:
- AI applications are standardized across teams  
- Governance and control are built-in  
- Data and access are centrally managed  
- Multi-model orchestration is simplified  

Peridot turns infrastructure into a usable system.

---

## When to choose AWS Bedrock vs Peridot

Choose AWS Bedrock if:
- You want raw model access  
- You have strong engineering resources  
- You are building from scratch  

Choose Peridot if:
- You want a platform for AI applications  
- You need governance and control  
- You want faster time to value  

---

## Punchline

AWS Bedrock gives you infrastructure.

Peridot gives you a system to build and control AI applications.
